# RailMate 团队介绍

## 1. 团队定位

RailMate 团队是一支兼具技术、产品与商业视角的跨学科创新小组，聚焦“AI + 交通出行”方向，目标是打造真正能落地、能被用户使用、也能在商业上成立的场景型智能产品。

团队的共同特点是：

- 既懂大模型，也懂真实用户场景；
- 既能写代码，也能设计产品闭环；
- 既关注创新亮点，也重视合规与商业化路径。

## 2. 核心分工

## 2.1 AI 技术负责人

### 职责

- 负责 RailMate Agent 架构设计
- 负责 Prompt 策略、工具调用、上下文记忆、规划模式
- 负责复杂场景能力：中转换乘、沿途游玩、买长乘短风险识别

### 背景优势

- 具备大模型应用开发和 Prompt Engineering 能力
- 熟悉 Agent、Function Calling、RAG 与场景化编排
- 能把复杂场景拆解成可执行的产品能力

## 2.2 后端工程负责人

### 职责

- 负责 `FastAPI` 服务框架、接口设计和数据层
- 负责车次查询、票价查询、数据同步、账号认证与任务调度
- 保障系统稳定性、模块可扩展性和演示闭环完整度

### 背景优势

- 熟悉 Python 后端开发与数据库建模
- 具备 API 设计、异步服务和工程化组织能力
- 能快速把比赛原型搭建成真实可运行系统

## 2.3 前端 / 交互负责人

### 职责

- 负责 Web 原型与多端体验设计
- 负责首页对话流、卡片展示、车次详情、设置页、行程页
- 负责语音输入、动态反馈和响应式布局实现

### 背景优势

- 熟悉现代前端工程栈与交互设计
- 能把复杂铁路信息转化为简洁直观界面
- 重视“科技感”和“可用性”的平衡

## 2.4 产品负责人

### 职责

- 负责需求定义、用户旅程梳理和比赛故事线设计
- 负责重点场景选择：语音输入、自然语言解析、顺路游玩、买长乘短等
- 负责文档输出、答辩包装与价值表达

### 背景优势

- 具备强场景抽象能力
- 擅长把复杂功能讲成清晰、有说服力的用户价值
- 能把技术亮点转化为评委易理解的产品语言

## 2.5 商业化 / 运营负责人

### 职责

- 负责市场分析、竞品研究与商业模式设计
- 负责会员订阅、B2B 输出、生态合作等路径规划
- 负责后续增长策略和用户运营思路

### 背景优势

- 具备商业洞察和增长思维
- 对交通、文旅和校园市场有较强敏感度
- 能帮助项目从“好 demo”走向“好产品”

## 3. 团队协作方式

团队采用“产品驱动 + 技术并行 + 商业前置”的协作模式：

1. 产品侧先定义高价值场景；
2. AI 与后端共同设计能力边界与接口；
3. 前端完成可展示、可操作的交互闭环；
4. 商业侧同步评估竞品与市场切入点；
5. 最终形成“可运行原型 + 可答辩故事 + 可商业化路径”的完整方案。

## 4. 团队优势总结

- **场景选得准**：铁路出行足够高频、真实、刚需。
- **技术方案硬**：AI Agent、FastAPI、中台化接口、多端适配都具备。
- **产品表达强**：不仅有功能，还有完整用户旅程。
- **商业意识足**：从一开始就思考会员、B2B、生态合作。
- **比赛完成度高**：可展示、可解释、可扩展。

## 5. 愿景

团队希望通过 RailMate 证明：AI 不只是聊天机器人，也可以成为真正进入日常生活、在复杂出行决策里帮用户省时间、省精力、降低焦虑的智能助手。

我们的目标不是做一个“会说话的查票工具”，而是做铁路出行时代的 AI 原生入口。
